# 球世界引力（面向完全不懂代码的人）

## 本章目标（你将学到什么）
- 理解“球形世界”的引力在游戏里是如何表现的（用生活化比喻）。
- 不需要编程基础，也能通过游戏内设置和实验感受不同重力的效果。
- 学会在哪里查看相关配置文件以及如何安全地做实验（不破坏原始存档或代码）。

## 用一句话说明
在“球形世界”里，引力总是朝向星球的中心（类似地球把一切拉向地心），项目中把这个方向视为“上”和“下”的参考方向，并用一个固定的数字来决定拉力的大小（不是随距离改变）。

## 直观比喻（帮助理解）
- 想象把一个小球放在大圆球的表面上：无论你站在大圆球的哪一边，你的脚都是朝着那个圆球的外表面——这就是“局部上方向”。
- 想象有一根绳子把你绑在星球上，绳子总是指向星球中心；把你往绳子方向拉的力就是“重力”。

## 项目里“上方向”是怎么决定的（不看代码也能懂）
- 游戏内部会记录“星球的中心在哪里”和“星球有多大”。
- 当游戏需要知道“上”是谁时，它会从玩家/物体当前位置，想象一条线连到星球中心，然后把这条线变成一个方向——这个方向就是“上”。

## 项目里重力具体表现的三个要点（非程序员版）
1. 方向是朝向星心：无论你站在哪个位置，重力总是把你往星球的中心拉。你在星球另一侧也会被拉“下去”。
2. 大小是一个固定值：游戏里有一个叫“重力加速度”的数字（例如默认约 19.62），这个数字决定你下落的快慢或跳得多高。改变这个数字会让整颗星球的重力变强或变弱。
3. 不随距离衰减：目前实现中，重力不会因为你离开星面一点点就变弱（没有按距离平方衰减的物理公式），它在可玩区域附近基本保持恒定。

## 我们如何在游戏里观察这些行为（零代码操作）
1. 在游戏的设置界面里可以找到一个“重力”滑块（Gravity），把它往左（变小）或往右（变大）移动，观察角色下落速度和跳跃高度的变化。这个滑块直接对应“重力加速度”的数值。
2. 你可以把星球半径设得很小（通过界面或存档里的配置），这会让“上”方向在空间中变化很快；观察相机是否平滑过渡，这帮助理解为什么代码里需要做“平滑处理”来避免镜头翻转。

## 安全的实验建议（非常重要）
- 永远在“副本”上实验：如果想修改存档里的设置，请先把 `saves/` 目录下对应的世界文件夹复制出来（例如 `saves/world_xxx` → `saves/world_xxx_copy`），在副本上改动并打开它验证效果。
- 不要直接修改 `src/` 下的源代码（除非你在一个独立的代码副本或分支上）。如果希望尝试代码层面的改动，请先在电脑上把整个项目文件夹复制一份，或者在 Git 上创建一个新分支进行实验。

## 建议的步骤（为零基础的你一步步来）
1. 启动游戏并进入世界创建或设置界面。找到“Gravity（重力）”滑块，记下当前值（例如 19.62）。
2. 把重力改为更小（例如 4），创建/加载世界，记录下落时间与跳跃高度的感觉（手动计时或用手机录像）。
3. 再把重力改为更大（例如 40），重复观察并记录。对比三次体验：原始、轻重力、强重力。
4. 试着把星球半径改成很小的值（例如接近 8），在世界里走一圈观察相机视角变化是否平滑或有突变。

## 小结（非技术回顾）
- 本章最重要的两点：1) “上方向”由你和星心的连线决定；2) 重力大小是一个可调的常数，项目中目前没有按距离衰减。体验与调整这些参数可以帮助你直观理解游戏物理如何影响玩家感受。

## 图示（向量与流程）
下面这幅图说明了项目中“上方向（local_up）”与重力向量的关系，以及相机平滑与跳跃如何使用该方向：

```mermaid
flowchart LR
	subgraph World[世界]
		PC[Planet Center\n(星心)]
		P[Player Position\n(玩家位置)]
	end

	PC -->|dir = P - PC| DirVec((方向向量))
	DirVec -->|normalize| LocalUp[local_up\n(局部上向量)]
	LocalUp -->|gravity = -local_up * g| Gravity[Gravity\n(重力向量)]

	P -->|uses local_up for| Camera[Camera Up\n(相机上向量)]
	Camera -->|smooth lerp| SmoothedUp[Smoothed Up\n(平滑后的相机上向量)]

	P -->|jump impulse along| Jump[Jump Impulse\n(跳跃冲量)]
	LocalUp --> Jump

	style PC fill:#f9f,stroke:#333,stroke-width:1px
	style P fill:#bbf,stroke:#333,stroke-width:1px
	style LocalUp fill:#bfb,stroke:#333,stroke-width:1px
	style Gravity fill:#fbb,stroke:#333,stroke-width:1px
	style Camera fill:#ffd580,stroke:#333,stroke-width:1px

	%% 备注
	click LocalUp "" "local_up：由玩家位置减去星心向量并归一化得到"
![Planet Gravity Diagram](docs/assets/planet_gravity_diagram.svg)

```

图示说明：
- 从玩家位置指向星心得到一个向量（Dir），把它单位化得到 `local_up`。
- 重力向量就是 `-local_up`，再乘以一个标量 `g`（重力强度），这个 `g` 是可调的数值（UI 中的 Gravity 滑块）。
- 相机会用 `local_up` 作为“上”方向，但为了避免在小星球上快速翻转，会把上向量做平滑处理（lerp/指数平滑）。
- 跳跃方向以 `local_up` 为基准，把向上速度沿 `local_up` 分量增加。
